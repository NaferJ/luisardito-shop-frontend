## 🎯 Decisiones de Diseño

### ¿Por qué TypeScript?
- Type safety para notificaciones
- Intellisense en el IDE
- Mejor mantenibilidad
- Menos bugs en producción

### ¿Por qué Chakra UI?
- Ya instalado en el proyecto
- Componentes accesibles
- Temas light/dark integrados
- Responsive grid system

### ¿Por qué 30 segundos de refresh?
- Balance entre:
  - Datos actualizados (no 5 minutos)
  - Bajo overhead (no 5 segundos)
  - UX acceptable
- Para tiempo real, agregar WebSockets

### ¿Por qué solo desktop?
- En móvil cabe en drawer después
- Prioridad inicial: desktop
- Escalable a móvil fácilmente

## ❓ FAQ

**P: ¿Necesito hacer algo más para que funcione?**
R: Solo ejecutar la migración en backend: `npm run migrate`

**P: ¿Funciona sin WebSockets?**
R: Sí, polling cada 30s es suficiente para la mayoría.

**P: ¿Puedo customizarlo?**
R: Sí, hay ejemplos en `EJEMPLOS_NOTIFICACIONES.md`

**P: ¿Funciona en móvil?**
R: Actualmente está oculto en navbar móvil, pero el modal abre bien.

**P: ¿Es accesible?**
R: Sí, tiene aria-labels, tooltips, y contraste WCAG.

**P: ¿Necesito cambiar la API?**
R: No, solo ejecutar la migración. La API ya genera notificaciones.

**P: ¿Qué pasa con las notificaciones antiguas?**
R: Se guardan soft-deleted. Puedes limpiarlas en background.

**P: ¿Es seguro?**
R: Sí, todo requiere autenticación JWT.

## 📞 Soporte Rápido

### Si algo no funciona:
1. Abre DevTools (F12)
2. Network → filtra "notificaciones"
3. Verifica status code (200 = ok, 401 = auth)
4. Console → busca errores rojo
5. Revisa `.env.local` → `NEXT_PUBLIC_API_URL`

### Si tienes dudas:
1. Busca en la documentación (usa Ctrl+F)
2. Mira los ejemplos en `EJEMPLOS_NOTIFICACIONES.md`
3. Revisa checklist en `CHECKLIST_PROXIMOS_PASOS.md`
